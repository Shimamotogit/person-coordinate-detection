# Stand & Answer - Camera AI Quiz

カメラで登録した参加者を姿勢推定で追跡し、画面上の回答ゾーンへ移動することでクイズに回答できるブラウザアプリです。

## 主な機能

- スタート画面で出題数と最大点を同じ画面から設定
- 全問終了後に得点と正解数を表示する終了画面
- 問題・カメラ領域だけを全画面表示
- 全画面表示中はページ下部の「遊び方」を非表示
- 複数のカメラから使用するカメラを選択
- 複数人が映っている状態から参加者1人を登録
- MediaPipe Pose Landmarker Fullによる33点の姿勢ランドマーク検出
- 姿勢特徴、身体サイズ、移動方向を組み合わせた参加者追跡
- 人物を矩形で囲まず、参加者の頭上にポイントだけを表示
- 背後の人物へ追跡対象を自動で切り替えにくい設計
- 時間経過またはスペースキー後3秒で回答確定
- 2択以上の可変長選択肢とランダム出題
- カメラ映像をサーバーへ送信しないブラウザ内処理

## 起動方法

カメラAPIと `fetch()` を利用するため、`file://` で直接開かず、ローカルHTTPサーバーから起動してください。

```bash
python -m http.server 8080
```

ブラウザで `http://localhost:8080` を開きます。

> 本番公開時は HTTPS が必要です。localhost は多くのブラウザで安全なコンテキストとして扱われます。

## ゲームの流れ

1. スタート画面で出題数と最大点を設定します。
2. 使用するカメラを選択します。
3. 映像内の丸い番号から参加者1人を登録します。
4. 回答確定方法を選択してクイズを開始します。
5. 指定した問題数が終わると終了画面に得点と正解数を表示します。

## 点数配分

最大点を出題数で整数配分します。割り切れない場合は一部の問題を1点高くして、全問正解時の合計が指定した最大点と完全に一致するようにします。

例：3問、最大100点の場合は `34点・33点・33点` です。

## 全画面表示

クイズ画面の「全画面表示」を押すと、カメラ・問題・回答エリアを含むステージだけを全画面表示します。ページ下部の遊び方は表示されません。Escキーまたは「全画面を終了」で戻れます。

## 参加者追跡

従来のCOCO-SSDによる人物矩形検出から、MediaPipe Pose Landmarker Fullへ変更しました。Pose Landmarkerは1人につき33個の身体ランドマークを返すため、単一の矩形よりも姿勢、足元、身体サイズを安定して取得できます。

登録後は以下を組み合わせて同一人物を追跡します。

- 腰・肩を中心とした移動位置と移動方向
- 身体の見かけ上の高さと大きさ
- 3D姿勢ランドマークから計算した腕・脚・胴体の比率
- ランドマークの可視性

短時間検出が途切れても約4.5秒は再取得を試みます。別人への誤切替を避けるため、特徴が大きく異なる人物は採用しません。完全な遮蔽や参加者同士が長時間重なる状況では再登録が必要になる場合があります。

## 回答確定方法

### 時間で確定

登録者の足元が同じ回答エリアに設定時間以上留まると回答を確定します。設定画面で0.5〜10秒を指定できます。

### スペースキーで確定

管理者がスペースキーを押すと3秒のカウントダウンを開始し、終了時点で登録者が立っている回答エリアを採点します。

## 問題の追加・編集

`questions.json` を編集します。

```json
{
  "id": "sample",
  "text": "問題文",
  "choices": ["選択肢A", "選択肢B", "選択肢C"],
  "correctIndex": 1
}
```

- `choices`: 2個以上の可変長配列
- `correctIndex`: 正解の選択肢番号（0始まり）

共通設定：

```json
{
  "settings": {
    "holdDurationMs": 1200,
    "resultDisplayMs": 1300,
    "spaceCountdownSeconds": 3,
    "defaultQuestionCount": 5,
    "defaultMaxScore": 500
  }
}
```

## 使用モデル

- JavaScriptライブラリ: `@mediapipe/tasks-vision` 0.10.35
- モデル: Pose Landmarker Full float16
- 実行モード: VIDEO
- 最大検出人数: 6人
- GPUが利用できない場合はCPUへ自動フォールバック

モデルとWASMは初回起動時にCDNおよびGoogleのモデル配信先から読み込みます。

## 注意事項

- カメラから参加者の頭から足まで映るように配置してください。
- 参加者登録後はカメラ位置を動かさないでください。
- 暗い場所や人物が小さい場合は検出精度が下がります。
- 初回起動時はインターネット接続が必要です。
- MediaPipeの映像推論はメインスレッドで同期実行されるため、低性能端末ではフレームレートが低下する場合があります。
