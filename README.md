# カメラAI立ち位置クイズ

カメラ映像から登録した参加者を検出し、画面上の回答エリアへ移動して答えるブラウザ向けクイズアプリです。

静的なHTML・CSS・JavaScriptだけで動作します。カメラ映像、人物検出、採点、選択した問題JSONの検証と保存はブラウザ内で行い、選択したJSONやカメラ映像をサーバーへ送信しません。

## 主な機能

- JSONファイルから問題を読み込み
- JSONを選択しない場合は同梱のサンプル問題を使用
- 選択したJSONをブラウザ内のIndexedDBへ保存
- スタート画面で出題数と最大点を設定
- 登録問題から毎回ランダムに抽選して出題
- 「同じ設定でもう一度」でも再度ランダム抽選
- 全問終了後に得点と正解数を表示
- 複数のカメラから使用するカメラを選択
- 複数人の中から参加者1人を登録
- 高精度な人物物体検出モデルによる参加者検出
- 全身が映っていなくても、人物の横方向の中心から回答位置を判定
- 時間経過またはスペースキー操作による回答確定
- カメラ映像と問題データを外部へ送信しないブラウザ内処理

## 起動方法

カメラ機能、JavaScriptモジュール、サンプル問題の読み込みを利用するため、HTMLファイルを直接開かず、ローカルHTTPサーバーから起動してください。

```bash
python -m http.server 8080
```

ブラウザで次のアドレスを開きます。

```text
http://localhost:8080
```

本番環境へ公開する場合はHTTPSを使用してください。

## 問題JSONの使い方

開始画面の「JSONファイルを選択」から、ローカルにあるJSONファイルを選択します。

- ファイルはブラウザの`File` APIで読み取ります。
- 内容はブラウザ内で検証し、必要な項目だけを再構築します。
- 検証後のデータはブラウザ内のIndexedDBへ保存します。
- JSONをサーバーへ送る`fetch`、`XMLHttpRequest`、フォーム送信は行いません。
- 「サンプル問題に戻す」を押すと、ブラウザ内に保存した問題を削除して同梱の`questions.json`へ戻ります。

JSONを選択しない場合は、最初からサンプル問題が使用されます。

### JSON全体の例

```json
{
  "settings": {
    "holdDurationMs": 1200,
    "resultDisplayMs": 1300,
    "spaceCountdownSeconds": 3,
    "defaultQuestionCount": 5,
    "defaultMaxScore": 500
  },
  "questions": [
    {
      "id": "q1",
      "text": "日本の首都はどこ？",
      "choices": ["大阪", "東京", "福岡"],
      "correctIndex": 1
    }
  ]
}
```

### 問題項目

- `id`：問題を識別する文字列。省略した場合は自動で付与します。
- `text`：問題文
- `choices`：2〜20個の選択肢
- `correctIndex`：正解の選択肢番号。0から数えます。

### 設定項目

設定項目は省略できます。

- `holdDurationMs`：時間確定までのミリ秒。500〜10000
- `resultDisplayMs`：採点結果を表示するミリ秒。300〜10000
- `spaceCountdownSeconds`：スペース確定時の秒数。1〜10
- `defaultQuestionCount`：初期出題数。登録問題数以下
- `defaultMaxScore`：初期最大点。出題数以上、100000以下

## JSONの安全確認

ブラウザが重くなったり、不正なデータが処理へ入り込んだりすることを防ぐため、次の制限を設けています。

- ファイル形式：`.json`
- 最大ファイルサイズ：10MB
- 最大問題数：10000問
- 問題文：1000文字以内
- 選択肢：1項目500文字以内、1問あたり2〜20個
- 問題ID：120文字以内、重複不可
- 不明な項目は保存せず、必要な項目だけを新しいオブジェクトへコピー
- 問題文と選択肢は`textContent`で表示し、HTMLとして解釈しない

問題数の上限表示は、検証後の`questions`配列の件数から自動的に設定されます。

## ローカル処理と通信

### ブラウザ内で処理するもの

- 選択したJSONの読み取り、構文解析、検証
- 問題データの保存と読み出し
- カメラ映像
- 人物検出と追跡
- 回答判定と採点
- ランダム抽選

### 外部から読み込むもの

現在の構成では、MediaPipeのJavaScript・WASMと人物検出モデルをCDNからダウンロードします。ダウンロードしたモデルによる推論自体はブラウザ内で行います。

選択したJSONやカメラ映像を、CDNやモデル配信先へ送信する処理はありません。

完全にオフライン化する場合は、MediaPipeの配布ファイル、WASM、モデルファイルを同じサイト内へ配置し、`app.js`と`detector-compat.js`の参照先をローカルパスへ変更してください。

## セキュリティ対策

- Content Security Policyを`index.html`に設定
- 外部通信先をMediaPipe配信元とモデル配信元へ限定
- `object-src 'none'`、`base-uri 'none'`、`form-action 'none'`を設定
- リファラー送信を無効化
- JSONは許可した項目だけを再構築
- ファイルサイズ、文字数、選択肢数、問題数を検証
- IndexedDBから読み出したデータも再検証
- `_headers`対応の静的ホスティングでは、クリックジャッキング防止、MIMEスニッフィング防止、不要なブラウザ機能の無効化を追加

ホスティングサービスが`_headers`に対応していない場合は、同等のHTTPレスポンスヘッダーをサービス側の設定で追加してください。

## ゲームの流れ

1. 必要に応じてJSONファイルを選択します。
2. 出題数と最大点を設定します。
3. 使用するカメラを選択します。
4. 映像内の丸い番号から参加者1人を登録します。
5. 回答の確定方法を選択してクイズを開始します。
6. 指定した問題数が終わると、終了画面に得点と正解数を表示します。

## ランダム出題

ゲームを開始するたびに全問題をシャッフルし、設定した出題数だけ抽選します。

「同じ設定でもう一度」を押した場合も、新しいゲームとして再度シャッフルします。乱数の結果として、前回と同じ問題や順番になる可能性はあります。

## 人物検出

骨格推定ではなく、人物の物体検出を使用しています。

顔から上半身だけが映っている場合や、足が画面外にある場合でも、人物として検出できれば動作します。回答エリアは人物検出範囲の横方向の中心で判定します。

GPU利用時はEfficientDet-Lite2のfloat16版、CPU利用時はint8版を使用します。GPU初期化に失敗した場合は自動的にCPUへ切り替わります。

## 回答の確定方法

### 時間で確定

登録者が同じ回答エリアに設定時間以上留まると回答を確定します。採点結果を表示した後、自動的に次の問題へ進みます。

### スペースキーで確定

1. 管理者がスペースキーを押すと、3秒のカウントダウンを開始します。
2. カウントダウン終了時の位置で回答を採点します。
3. 採点後はその画面で停止します。
4. 管理者がもう一度スペースキーを押すと、次の問題へ進みます。
5. 最終問題の採点後も、スペースキーを押してから結果画面へ進みます。

## ファイル構成

```text

├── index.html                 画面構成とCSP
├── styles.css                全体の見た目
├── question-source.css       JSON選択画面の見た目

├── question-source.js        JSONのローカル読み込み・検証・保存
├── detector-compat.js        人物検出の互換修正
├── app.js                    クイズと追跡の主要処理
├── questions.json            未選択時のサンプル問題
├── _headers                   対応ホスティング向けセキュリティヘッダー

```

## うまく人物を検出できない場合

1. ブラウザでカメラの利用を許可しているか確認します。
2. 人物の頭部と胴体が画面内に入るようにします。
3. 強い逆光を避け、人物の正面から光が当たるようにします。
4. ブラウザを再読み込みします。
5. 古いファイルが残っている場合は強制再読み込みします。

- Windows：`Ctrl + F5`
- macOS：`Command + Shift + R`

## 注意事項

- 参加者登録後はカメラ位置を動かさないでください。
- 参加者と背景の人物が長時間完全に重なる場合は、再登録が必要になることがあります。
- 暗い場所、強い逆光、人物が極端に小さい場合は検出精度が下がります。
- 高精度モデルを使うため、低性能端末では映像の動きが遅くなる場合があります。
- ブラウザのサイトデータを削除すると、IndexedDBに保存したローカル問題も削除されます。
