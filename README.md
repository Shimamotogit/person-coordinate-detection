# Stand & Answer - Camera AI Quiz

カメラ映像から登録した参加者を検出し、画面上の回答ゾーンへ移動して回答するブラウザアプリです。

## 主な機能

- スタート画面で出題数と最大点を設定
- 全問終了後に得点と正解数を表示
- カメラ・問題領域だけを全画面表示
- 複数カメラから使用するカメラを選択
- 複数人の中から参加者1人を登録
- EfficientDet-Lite2による高精度な人物物体検出
- 全身が映っていなくても、人物矩形の横方向中央から回答位置を判定
- 登録時の服装色、位置、移動方向、矩形サイズを使った追跡
- 人物を矩形で囲まず、参加者の頭上にポイントだけを表示
- 時間経過またはスペースキー操作による回答確定
- 2択以上の可変長選択肢とランダム出題
- カメラ映像をサーバーへ送信しないブラウザ内処理

## 起動方法

カメラAPIと `fetch()` を利用するため、ローカルHTTPサーバーから起動してください。

```bash
python -m http.server 8080
```

ブラウザで `http://localhost:8080` を開きます。

> 本番公開時は HTTPS が必要です。localhost は多くのブラウザで安全なコンテキストとして扱われます。

## ゲームの流れ

1. スタート画面で出題数と最大点を設定します。
2. 使用するカメラを選択します。
3. 映像内の丸い番号から参加者1人を登録します。
4. 回答確定方法を選択してクイズを開始します。
5. 指定した問題数が終わると終了画面に得点と正解数を表示します。

## 全身が映っていない場合

骨格推定ではなく人物の物体検出を使っています。顔から上半身だけ、または足が画面外の状態でも、モデルが人物として検出できれば動作します。

回答ゾーンは人物矩形の横方向中央を使って判定します。足元が見えている必要はありません。安定性を上げるため、少なくとも頭部と胴体の大部分が映る配置を推奨します。

## 参加者追跡

人物検出には MediaPipe Object Detector と EfficientDet-Lite2 int8モデルを利用します。EfficientDet-Lite2は448×448入力を使い、軽量モデルより精度を優先した構成です。

登録後は次の情報を組み合わせて同一人物を追跡します。

- 現在位置と直前までの移動方向
- 検出矩形の重なりとサイズ変化
- 登録時の服装を含む簡易的な色特徴
- 人物検出の信頼度

短時間検出が途切れても約5.2秒は再取得を試みます。別の人物への誤切替を防ぐため、見た目や移動位置が大きく異なる人物は登録者として採用しません。

## 回答確定方法

### 時間で確定

登録者が同じ回答エリアに設定時間以上留まると回答を確定します。採点結果の表示後は自動的に次の問題へ進みます。

### スペースキーで確定

1. 管理者がスペースキーを押すと3秒カウントダウンを開始します。
2. カウントダウン終了時の位置で採点します。
3. 採点後は画面を停止し、スペースキーがもう一度押されるまで次の問題へ進みません。
4. 最終問題の採点後も、スペースキーを押してから結果画面へ進みます。

## 点数配分

最大点を出題数で整数配分します。割り切れない場合は一部の問題を1点高くして、全問正解時の合計が指定した最大点と一致するようにします。

例：3問、最大100点の場合は `34点・33点・33点` です。

## 問題の追加・編集

`questions.json` を編集します。

```json
{
  "id": "sample",
  "text": "問題文",
  "choices": ["選択肢A", "選択肢B", "選択肢C"],
  "correctIndex": 1
}
```

- `choices`: 2個以上の可変長配列
- `correctIndex`: 正解の選択肢番号（0始まり）

## 使用モデル

- JavaScriptライブラリ: `@mediapipe/tasks-vision` 0.10.35
- タスク: MediaPipe Object Detector
- モデル: EfficientDet-Lite2 int8
- 入力サイズ: 448×448
- 対象カテゴリ: `person`
- 実行モード: `VIDEO`
- GPU優先、利用できない場合はCPUへフォールバック

モデルとWASMは初回起動時にCDNおよびGoogleのモデル配信先から読み込みます。

## 注意事項

- 人物の頭部と胴体がある程度映るようにしてください。
- 参加者登録後はカメラ位置を動かさないでください。
- 参加者と背景の人物が長時間完全に重なる場合は再登録が必要になることがあります。
- 暗い場所、強い逆光、人物が極端に小さい場合は検出精度が下がります。
- EfficientDet-Lite2は精度を優先するため、低性能端末ではフレームレートが下がる場合があります。
