# カメラAI立ち位置クイズ

カメラ映像から登録した参加者を検出し、画面上の回答エリアへ移動して答えるブラウザ向けクイズアプリです。

## 主な機能

- スタート画面で出題数と最大点を設定
- 全問終了後に得点と正解数を表示
- カメラ・問題・回答エリアだけを全画面表示
- 複数のカメラから使用するカメラを選択
- 複数人の中から参加者1人を登録
- 高精度な人物物体検出モデルによる参加者検出
- 全身が映っていなくても、人物の横方向の中心から回答位置を判定
- 登録時の服装色、位置、移動方向、検出範囲の大きさを使った追跡
- 人物を矩形で囲まず、参加者の頭上に得点だけを表示
- 時間経過またはスペースキー操作による回答確定
- 2択以上の可変長選択肢とランダム出題
- カメラ映像をサーバーへ送信しないブラウザ内処理

## 起動方法

カメラ機能と問題ファイルの読み込みを利用するため、HTMLファイルを直接開かず、ローカルHTTPサーバーから起動してください。

```bash
python -m http.server 8080
```

ブラウザで次のアドレスを開きます。

```text
http://localhost:8080
```

本番環境へ公開する場合はHTTPSが必要です。

## ゲームの流れ

1. スタート画面で出題数と最大点を設定します。
2. 使用するカメラを選択します。
3. 映像内の丸い番号から参加者1人を登録します。
4. 回答の確定方法を選択してクイズを開始します。
5. 指定した問題数が終わると、終了画面に得点と正解数を表示します。

## 全身が映っていない場合

骨格推定ではなく、人物の物体検出を使用しています。

顔から上半身だけが映っている場合や、足が画面外にある場合でも、人物として検出できれば動作します。回答エリアは人物検出範囲の横方向の中心で判定するため、足元が見えている必要はありません。

安定して利用するため、頭部と胴体の大部分が映るようにカメラを配置してください。

## 人物検出の互換修正

人物がまったく検出されない環境への対策として、次の修正を追加しています。

- GPU利用時はEfficientDet-Lite2のfloat16版を使用
- CPU利用時はEfficientDet-Lite2のint8版を使用
- 推論前のカテゴリ許可リストを使用せず、推論結果から人物カテゴリを判定
- 英語名、表示名、COCOカテゴリ番号のいずれでも人物を識別
- 部分的に映った人物を拾いやすいよう、人物検出のしきい値を調整

GPU初期化に失敗した場合は、自動的にCPU処理へ切り替わります。

## 参加者の追跡

参加者を登録した後は、次の情報を組み合わせて同じ人物を追跡します。

- 現在位置と直前までの移動方向
- 検出範囲の重なりと大きさの変化
- 登録時の服装を含む簡易的な色特徴
- 人物検出の信頼度

短時間だけ人物検出が途切れた場合は、すぐに再登録画面へ移動せず、一定時間同じ人物の再取得を試みます。見た目や移動位置が大きく異なる人物へは自動で切り替えません。

## 回答の確定方法

### 時間で確定

登録者が同じ回答エリアに設定時間以上留まると回答を確定します。採点結果を表示した後、自動的に次の問題へ進みます。

### スペースキーで確定

1. 管理者がスペースキーを押すと、3秒のカウントダウンを開始します。
2. カウントダウン終了時の位置で回答を採点します。
3. 採点後はその画面で停止します。
4. 管理者がもう一度スペースキーを押すと、次の問題へ進みます。
5. 最終問題の採点後も、スペースキーを押してから結果画面へ進みます。

## 点数配分

最大点を出題数で整数配分します。割り切れない場合は一部の問題を1点高くし、全問正解時の合計が指定した最大点と一致するようにします。

例として、3問で最大100点の場合は、`34点・33点・33点`です。

## 問題の追加と編集

`questions.json`を編集します。

```json
{
  "id": "sample",
  "text": "問題文",
  "choices": ["選択肢A", "選択肢B", "選択肢C"],
  "correctIndex": 1
}
```

- `choices`：2個以上の選択肢
- `correctIndex`：正解となる選択肢の番号。0から数えます。

## 使用している人物検出モデル

- ライブラリ：MediaPipe Tasks Vision 0.10.35
- 処理内容：人物の物体検出
- モデル：EfficientDet-Lite2
- 入力画像サイズ：448×448
- 対象カテゴリ：人物
- 処理モード：動画
- GPU利用時：float16版
- CPU利用時：int8版

モデルと実行用ファイルは、初回起動時に外部配信元から読み込みます。そのため、初回起動時はインターネット接続が必要です。

## うまく人物を検出できない場合

次の点を確認してください。

1. ブラウザでカメラの利用を許可しているか確認します。
2. 人物の頭部と胴体が画面内に入るようにします。
3. 強い逆光を避け、人物の正面から光が当たるようにします。
4. ブラウザを再読み込みします。
5. 古いファイルが残っている場合は、強制再読み込みを行います。

強制再読み込みの代表的な操作は次のとおりです。

- Windows：`Ctrl + F5`
- macOS：`Command + Shift + R`

## 注意事項

- 参加者登録後はカメラ位置を動かさないでください。
- 参加者と背景の人物が長時間完全に重なる場合は、再登録が必要になることがあります。
- 暗い場所、強い逆光、人物が極端に小さい場合は検出精度が下がります。
- 高精度モデルを使うため、低性能端末では映像の動きが遅くなる場合があります。
